# Development Status

Date: 2026-05-12

## Current Goal

Build a TypeScript/Node.js MCP server that analyzes Python projects, generates reusable static indexes under `.project-kg/`, and produces draw.io diagrams for agents helping users understand project structure and feature call flows.

## Confirmed MVP Scope

- Implementation language: TypeScript + Node.js.
- Target analysis language: Python.
- Parser: `tree-sitter` + `tree-sitter-python`.
- Index output directory in analyzed projects: `.project-kg/`.
- Diagram modes:
  - `core_modules`: core Python module dependency diagram.
  - `feature_trace`: bounded function/method call-chain diagram.

- Project scaffolding with `package.json`, `tsconfig.json`, `README.md`, `.gitignore`, and updated `AGENTS.md`.
- MCP server entrypoint in `src/index.ts` and server wiring in `src/mcp/`.
- Tool handlers:
  - `build_project_index`

## Current Limitations

- Import resolution is basic. It handles simple module-to-file candidates but needs better support for package roots, `from app.x import Y`, and relative imports.
- Call graph resolution is simple name-based. It does not yet infer object types, so calls such as `self.repository.find_by_email()` may not resolve to the exact class method.
- `feature_trace` currently works best when queried by exact function, method, class, qualified name, or file-path fragment.
- draw.io layout is a simple grid, not a dependency-aware hierarchical layout.
- No CLI wrapper exists yet for invoking tools outside MCP.

## Suggested Next Steps

1. Improve Python import resolution using package roots and `__init__.py` discovery.
2. Improve method-call resolution for `self.x.method()` and imported class instances.
3. Add integration tests for `buildProjectIndex`, `traceFeature`, and `generateDrawio`.
4. Add a small CLI or script to manually run indexing against a sample Python project.
5. Improve draw.io layout for `core_modules` with layered dependency positioning.
6. Add MCP client configuration examples to `README.md`.
